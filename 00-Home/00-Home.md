## 🗂️ How this vault is organized

Numbered folders read top-to-bottom as the project flows: **research → build → present.**

- `00-Home` — this dashboard (the map of everything)
- `01-Project` — charter, rules, timeline, the requirements tracker, roles
- `02-Brand-Positioning` — the "why Terra is different" thinking (25% of score)
- `03-Design-System` — the official TDV design system + brand kit (relocated, intact)
- `04-Product` — sitemap, IA, per-page specs, UX notes
- `05-Build` — tech stack, repo setup, deployment & submission mechanics
- `06-Claude-Playbook` — the Claude usage log + prompt library (25% of score)
- `07-Presentation` — Demo Day plan & slide outline (15% of score)
- `08-Resources` — reference PDFs, inspiration teardowns
- `09-Logs` — daily standup / decision logs
- `99-Archived` — superseded material (never delete, move here)
- `_templates` — note templates (see [[_templates/Page Spec Template|Page Spec]], [[_templates/Daily Log Template|Daily Log]])
